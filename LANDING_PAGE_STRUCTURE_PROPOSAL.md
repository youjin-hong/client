# AUTA 랜딩페이지 구조 제안서

## 📋 개요

포스터 내용과 랜딩페이지 베스트 프랙티스를 결합하여 **전환율 최적화**와 **발표 효과**를 동시에 달성하는 구조를 제안합니다.

---

## 🎯 랜딩페이지 3대 핵심 요소

1. **Hero Section** - 첫인상과 즉각적인 가치 제시
2. **정보 제공 섹션** - 신뢰 구축과 이해도 향상
3. **CTA Section** - 행동 유도와 전환 완성

---

## 📐 제안하는 섹션 구조

### **1. Hero Section (Above the Fold)**
**목적**: 즉각적인 관심 유도 + 핵심 가치 제시

**구성 요소**:
- ✅ **헤드라인**: "QA 비용은 절감하고, 속도는 높여주는 Figma 기반 UI 테스트 자동화"
- ✅ **서브헤드라인**: 문제 제기 + 해결책 제시 (포스터 서론 반영)
- ✅ **주요 CTA 버튼**: "지금 무료로 시작하기"
- ✅ **보조 CTA**: "5분 데모 보기" (비디오 링크)
- ✅ **핵심 지표 시각화**: 
  - 모델 성능 (mAP, Precision, Recall) - 막대 그래프
  - 효율 비교 (시간, 커버리지) - 막대 그래프
- ✅ **대시보드 프리뷰 캐러셀**: 실제 제품 화면 미리보기
  - 대시보드
  - 프로젝트 상세 페이지
  - 테스트 상세 페이지

**포스터 반영**: 서론의 문제 제기 + 성능지표의 핵심 수치

---

### **2. Problem & Solution Section (문제 제기 → 해결책)**
**목적**: 공감대 형성 + AUTA의 차별점 강조

**구성 요소**:
- **문제 상황** (포스터 서론 반영):
  - 수동 UI/UX 테스트의 비효율성
  - 코드 기반 자동화 도구의 디자이너 접근성 문제
- **AUTA의 해결책**:
  - Figma JSON + AI 시각 분석 (YOLO)
  - 클릭 한 번으로 드리프트 탐지
  - 페이지당 평균 2분, QA 비용 80% 이상 절감

**포스터 반영**: 서론의 핵심 내용

---

### **3. Technical Features Section (기술적 특이점)**
**목적**: 기술적 우위와 차별화 포인트 제시

**구성 요소**:
- **비교 테이블**: "기존 테스트 도구 VS AUTA"
  - 테스트 단위: DOM 요소 vs 시각적 컴포넌트 + UX 플로우
  - 입력 데이터: 코드 스크립트 vs Figma JSON (플러그인)
  - 분석 방식: 위치/이벤트 단일 기준 vs YOLO + OCR + LLM 종합 분석
  - UX 평가: 불가능 vs 시각화 리포트 가능
- **UI 테스트 설명**:
  - Routing (화면 이동)
  - Interaction (화면 내 변화)
  - Component Matching (컴포넌트 일치)
- **UX 테스트 설명**:
  - Nielsen 10원칙 기반 평가
  - LLM을 통한 사용자 경험 품질 평가
  - 시각적 박스로 UX 약점 표시

**포스터 반영**: 기술적 특이점 섹션 전체

---

### **4. Matching Example Section (매칭 예시)**
**목적**: 실제 작동 방식의 시각적 증명

**구성 요소**:
- **시각적 예시 갤러리**:
  - 디자인 vs 구현 비교 이미지
  - 오류 탐지 결과 시각화
  - 컴포넌트 매핑 이슈 표시
- **인터랙티브 데모** (선택사항):
  - Before/After 슬라이더
  - 오류 발견 과정 애니메이션

**포스터 반영**: 매칭 예시 섹션

---

### **5. How It Works Section (작동 구조)**
**목적**: 간단하고 명확한 프로세스 설명

**구성 요소**:
- **3단계 프로세스** (포스터 작동구조 반영):
  1. **Figma 페이지 → Plugin JSON 추출**
     - Figma 플러그인 인터페이스 이미지
  2. **UI 컴포넌트 탐지 & OCR 텍스트 추출**
     - YOLO 탐지 결과 이미지 (바운딩 박스)
  3. **오류 발견 및 저장**
     - 오류 하이라이트 이미지 + 리포트 사이드바

**포스터 반영**: 작동구조 섹션의 3단계

---

### **6. Performance Indicators Section (성능 지표)**
**목적**: 검증된 데이터로 신뢰 구축

**구성 요소**:
- **테스트 효율 비교** (포스터 성능지표 반영):
  - 소요 시간: 기존 QA 30분 vs AUTA 2분
  - 테스트 커버리지: 화면 단위 일부 vs 전체 UI 단위 커버
  - 실행 인원: 최소 1명 vs 0명 (자동화)
- **모델 성능 지표**:
  - mAP@50-95: 57.8
  - mAP@50: 70.6
  - Precision: 71.1
  - Recall: 67.1
  - 막대 그래프로 시각화

**포스터 반영**: 성능지표 섹션 전체

---

### **7. Video Tutorial Section (사용법 영상)**
**목적**: 실제 사용 경험 제공 + 진입 장벽 낮추기

**구성 요소**:
- **유튜브 영상 임베드**: "5분 만에 AUTA 셋업 & 첫 테스트"
- **영상 설명**: 설치부터 리포트 해석까지 한 번에
- **CTA**: "가이드 보며 5분 설치"

**포스터 반영**: 작동 구조의 시각적 보완

---

### **8. Social Proof Section (신뢰 요소)**
**목적**: 추가 신뢰 구축 (선택사항)

**구성 요소**:
- 광운대학교 로고 및 프로젝트 정보
- 개발팀 소개 (오준혁, 송인섭, 최현준, 홍유진)
- 캡스톤 프로젝트 명시

**포스터 반영**: 헤더의 팀 정보

---

### **9. Final CTA Section (최종 행동 유도)**
**목적**: 전환 완성

**구성 요소**:
- **강력한 헤드라인**: "지금, 내 프로젝트로 정확히 검증해 보세요"
- **서브헤드라인**: "무료로 시작하고, 5분 안에 첫 테스트 완료"
- **주요 CTA 버튼**: "무료로 시작하기"
- **보조 CTA 버튼**: "가이드 보며 5분 설치"
- **긴급성/혜택 강조**: "무료", "5분", "즉시 시작"

**포스터 반영**: 결론의 장점 요약

## 🎤 발표를 위한 구조적 특징

1. **스토리텔링 흐름**:
   - 문제 제기 → 해결책 제시 → 작동 방식 → 성능 증명 → 행동 유도

2. **시각적 증거**:
   - 포스터의 모든 핵심 내용을 섹션으로 재구성
   - 실제 제품 화면 캐러셀로 구체성 확보

3. **데이터 중심**:
   - 모든 수치를 포스터 근거로 명시
   - 비교 테이블과 그래프로 객관성 강조

4. **인터랙티브 요소**:
   - 캐러셀, 애니메이션, 비디오로 참여도 향상
